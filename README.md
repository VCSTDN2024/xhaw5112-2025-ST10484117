## Project Overview

The **[Web Page and Mobile Application]** is a mobile application developed as part of an assignment in the [Xhaw] subject. This application was created using Figma. The app's primary purpose is to The purpose of the "Empowering the Nation" app is to showcase training courses, receive inquiries, and provide quotes, making it easy for potential students to learn about and access skills development programs.

The app was developed to meet the requirements of the assignment, which includes creating a functional mobile app and utilizing GitHub for version control and CI/CD automation using GitHub Actions.

---

## Purpose and Features

### Purpose:
The main goal of this app is to simplify course advertising and inquiry management for "Empowering the Nation," making it easy for people to find and apply for training. 

### Key Features:
- Feature 1: Course Information and Enrollment: Displays all six-month and six-week courses with details on fees, purpose, and content.
- Feature 2: Inquiry and Quote Request: Allows users to submit inquiries about courses and request personalized quotes, with automated discount calculations.
- Feature 3: About Us Section: Provides information on the training school's history, mission, and Precious Radebe's motivation, including testimonials.


## Design Considerations

The design of the **[Web Page and Mobile Application Report ]** was based on the following key considerations:

1. **User Experience (UX)**: The app was designed with the goal of providing an intuitive and easy-to-navigate user interface, ensuring a seamless user experience.
   
2. **Responsiveness**: The app was built to ensure it performs well on different screen sizes, with particular attention to device compatibility.
   
3. **Simplicity**: The design is minimalistic, focusing on core features without overwhelming the user.
   
4. **Performance**: Code optimization was considered to ensure fast response times and low battery usage.

## GitHub and GitHub Actions

This project was managed using **GitHub** for version control, where all code changes were committed and pushed regularly. GitHub enabled collaborative coding, allowing me to keep track of changes and maintain project integrity.

### GitHub Actions:
I utilized **GitHub Actions** to automate the build and deployment process. This includes:

- Running automated **tests** to ensure the app’s functionality.
- Compiling the app into **APK** and **AAB** files, which are the formats required for distribution.
- Uploading these build artifacts to GitHub for easy access.

The workflow ensures that my project is automatically built and tested every time I push changes, and it simplifies the process of delivering the final APK/AAB files for submission.
